# Business Product Publish

用于商品发布、URL 发品、素材包发品、商品查询、商品信息优化、批量编辑。候选域：`product.icbu-product-agent`。

## 意图路由

- 用户提供 1688、淘宝、天猫、AliExpress、Amazon 等商品链接：URL 发品。
- 用户上传图片、ZIP、Excel、Word 等素材：素材包发品。
- 查询商品、质量分、类目、属性、草稿、任务：商品管理查询。
- 优化标题、卖点、属性、SKU、图片信息：先查商品信息，再生成建议。
- 发布、直发、批量编辑、批量优化：写操作，必须确认。

## URL 发品顺序

1. 提取并校验 URL，去重，确认数量。
2. 查 `precheck-url-product-generate` schema，执行预检。
3. 预检失败则展示原因，停止。
4. 展示待发品 URL、发布类型、数量和影响；用户确认后启动。
5. 启动后优先 `--wait --poll-interval 5s --wait-timeout 30m`；需要后台提交时用 `--async`，再 `task wait/status` 恢复。
6. 展示草稿/发布结果和可恢复任务 ID。

## 素材包发品顺序

1. 确认素材文件可访问，必要时先完成素材解析。
2. 类目预测，展示候选类目，必要时让用户确认。
3. 属性/SKU 映射。
4. 用户修改内容时先合并素材，不要手写最终大 JSON。
5. 发品预检，失败则停止。
6. 用户确认后启动发品；优先 `--wait`，超时或后台执行时返回 `taskId/batch_id` 和恢复命令。

## 商品信息优化顺序

1. 通过商品 ID、名称、质量分、草稿状态等查询商品。
2. 查询类目属性、质量分、风险品牌词等辅助信息；多个商品或多个辅助查询互不依赖时用 batch 并行。
3. 生成优化建议，明确哪些字段会改。
4. 批量编辑或发布前，展示商品 ID、字段 diff、影响范围并取得确认。

## 注意事项

- 当前流程中的 `taskId`、cache key、material key 只能来自本轮上游返回，不要复用历史会话。
- 大 JSON 入参优先用 `@file` 或 stdin。
- 如果 schema/summary 标记废弃或禁止使用，即使命令存在也不要调用。
- 默认读顶层业务字段；缺失时再读 `.data.*` 或 schema 推荐 jq。
