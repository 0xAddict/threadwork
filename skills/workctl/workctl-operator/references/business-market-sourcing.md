# Business Market Sourcing

用于热品洞察、蓝海机会、Amazon/站外市场分析、1688/供应商/分销找品。候选域：`alibaba.product-selection-server`、`alibaba.supplier-verification-report`、`fulfillment`、`default` buyer-assistant、`cogito`。

## 意图路由

- “什么热卖/爆品/趋势”：热品洞察。
- “哪些蓝海/低竞争/高增长/供给不足”：蓝海机会。
- “找同款/找货/找供应商/1688”：找品或供应商链路。
- “验证供应商/看公司资质”：供应商验证。
- “分销找品/渠道商品/源商品/SKU 映射”：分销履约链路。

## 热品洞察顺序

1. 判断平台：国际站、Amazon、Temu、TikTok、SHEIN、web search 等。
2. 平台是排他选择；不要混用多条需求侧路径。
3. 国际站供给/热品通常先类目预测，再查热门商品或供给指标。
4. 类目确定后，热门商品、供给指标、价格带、趋势等只读数据可 batch 并行查。
5. 站外需求侧按平台查热销或市场数据；多个关键词/品类可并行但不要混平台口径。
6. 输出真实数据来源、窗口、Top 商品/品类和可行动建议。

## 蓝海机会顺序

1. 明确目标平台、品类、市场、价格带或工厂能力。
2. 并行获取供给侧和需求侧数据，优先用 `workctl batch call` 收敛到一个 artifact。
3. 做供需差、竞争强度、价格带、利润/物流/合规可行性判断。
4. 输出机会排序、证据、适合发品策略和风险。

## 供应商验证顺序

1. 用户给公司名或店铺 URL 时，先 recall 候选供应商。
2. 从候选中取 company ID。
3. 多个候选 company ID 需要比较时，用 batch 并行查 verification detail。
4. 输出公司、资质、风险、匹配可信度；候选不唯一时请用户确认。

## 分销找品顺序

1. 按关键词、图片、商品 ID 或渠道商品查询源商品。
2. 查 SKU 映射、运费、渠道关系。
3. 创建订单、批量导入、发货等写操作前确认渠道、商品、SKU、数量、地址、费用。
4. 查询物流或订单状态时使用上游订单号/物流单号。

## 注意事项

- 没有真实数据就不要输出排名、销量、价格、评分或链接。
- 报告中使用平台名和数据窗口，不暴露旧内部工具名。
- 1688 与 Alibaba.com 国际站不是同一供给侧；业务含义要分清。
- 商品、供应商、SKU、渠道订单号必须来自用户输入或工具返回。
