# Business Logistics Trade Risk

用于物流运费、关税计税、交易合同、拒付申诉、店铺/商品风险。候选域：`logistics`、`trade`、`fulfillment`。

## 意图路由

- 商品 ID 或商品链接 + 目的国 + 运费：商品运费。
- 重量/体积/计费重 + 目的国：线路或查价。
- 海关编码、税率、进口成本：关税计税。
- 订单拒付、申诉材料：拒付材料。
- 店铺风险、商品风险、侵权/品牌词：风险诊断。
- 合同/订单列表：交易列表或履约订单查询。

## 物流运费顺序

1. 判断是否有商品 ID。
2. 有商品 ID：查商品运费，补目的国、数量等必填字段。
3. 无商品 ID：收集重量、体积、目的国，查物流线路或报价。
4. 多个目的国、多个商品或多条线路是只读 fan-out，优先 batch 并行查。
5. 输出线路、费用、时效、限制条件；无数据时说明缺失字段。

## 关税计税顺序

1. 收集商品名、材质、用途、目的国、申报金额、币种、数量。
2. 查 schema，确认字段格式。
3. 执行归类和计税。
4. 输出 HS/海关编码、税费、置信度、需要人工确认的字段。

## 拒付和风险顺序

1. 拒付：提取订单 ID，查拒付材料；无数据则提示订单是否进入拒付流程。
2. 商品风险：提取商品 ID 或 URL，触发风险诊断。
3. 店铺风险：明确店铺范围后触发诊断。
4. 多个订单、商品、风险维度互不依赖时 batch 并行查询；申诉、创建、发货等写操作保持确认后串行。
5. 输出风险项、严重程度、后台处理入口或建议动作。

## 注意事项

- 国家、重量、商品 ID、订单 ID 是硬前置；缺失时先追问。
- 不要编造海关编码、税率、订单状态、物流价格。
- 创建订单、批量导入、发货属于高影响操作，必须确认。
- 如果工具返回业务错误，保留 trace/code/category/retryable，判断是否可重试。
