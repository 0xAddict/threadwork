# Snoopy SOP вАФ Auto-Recycle Playbook

This file is NOT a runtime prompt anymore. It is documentation of the
standing operating procedure Snoopy follows when an "AUTO-RECYCLE NUDGE"
arrives in his tmux pane from `~/.claude/hooks/context-budget-watch.sh`.

The hook composes a brief like:

    рЯЫОпЄП AUTO-RECYCLE NUDGE
    target_agent: <name>
    session_id: <uuid>
    iso_ts: <utc-iso8601>
    est_tokens: <int>
    budget: <int>
    test_mode: 0|1

    Run your auto-recycle SOP: ...

When you (Snoopy) see that block in your pane, execute the steps below in
order. Be terse. One pass. Then go idle.

## Inputs (parsed from the nudge body)

- AGENT     = `target_agent` (boss / steve / sadie / kiera)
- SESSION   = `session_id`
- ISO_TS    = `iso_ts`
- TOKENS    = `est_tokens`
- BUDGET    = `budget`
- TEST_MODE = `test_mode` (if 1, prefix every Telegram message with `[TEST] `)

## Steps

1. **Force debrief (boss only).** If `AGENT == "boss"`, call
   `mcp__task-board__force_debrief`. For non-boss targets, skip вАФ per-agent
   handoff in step 3 is sufficient.

2. **Enumerate active work.** Call
   `mcp__task-board__list_tasks(filter="mine", agent=AGENT)` to get the
   target's open task IDs and one-line descriptions.

3. **Save the handoff memory.** Call `mcp__task-board__save_memory` with:
   - `agent`: AGENT
   - `category`: "fact"
   - `importance`: 5
   - `pinned`: false
   - `classification`: "operational"
   - `content`: a markdown block beginning EXACTLY with the line
     `[session-handoff:<AGENT>:<ISO_TS>]` (substitute values), followed by:

         ACTIVE TASKS: <comma-separated open task IDs with one-liners, or "none">
         IN-FLIGHT DECISIONS: <list_decisions if quick; else "unknown вАФ check on rehydrate">
         RECENT LEARNINGS: auto-recycle fired at <TOKENS>/<BUDGET> tokens; session <SESSION>
         CHILD SUBAGENTS: <get_children for active tasks, or "none visible">
         NEXT ACTION: Resume the highest-priority active task above after rehydrate.

   The `[session-handoff:<agent>:` prefix is what `session-boot.sh` greps
   for. Do not deviate from the prefix shape.

   вЪ† VERIFY THE ROW AFTER SAVING (2026-07-11 field finding): save_memory
   stores the row under the CALLER (snoopy) with state=proposed, clamps
   importance to 3, and a sanitizer injects вЯ¶escвЯІ into the bracket prefix вАФ
   all four break session-boot.sh's lookup (it requires agent=<target> or
   'shared', state='active', and content LIKE '[session-handoff:<target>:%').
   Fix via direct sqlite UPDATE on memories (agent, state, importance,
   content de-вЯ¶escвЯІ), then prove it by running session-boot.sh's exact
   SELECT with LABEL=<target>. Never trust the handoff until that query
   returns the row.

4. **Inject `/clear` into the target's pane.** Use Bash:

       tmux send-keys -t claude-<AGENT> '/clear' Enter

   This is the persistent-agent path вАФ only Snoopy does this; the hook
   itself never types `/clear` into another session. If the target session
   doesn't exist or send-keys errors, note it and continue to step 5; the
   target's own banner output told them to type `/clear` manually.

   вЪ† IF THE TARGET IS PIPELINING (2026-07-11): when the target spawns lane
   waves back-to-back, a natural idle gap may never arrive before
   auto-compaction. Don't just poll вАФ send a nudge_agent CHECKPOINT REQUEST:
   "when current lanes complete, checkpoint to board/disk, HOLD new spawns,
   go idle вАФ idle IS the ack." Boss honored this first try and self-declared
   "Ready for snoopy's /clear." Coordinated beats racing compaction.
   REFINEMENT (2026-07-12): if the target DECLINES the hold because a human
   is live-directing work, don't fight priority вАФ negotiate a DECLARED
   boundary instead ("after these lanes land / after the wave ships") plus
   a hard token line (~750k) as the override, and have the target stage the
   pickup as a board note and announce "idle at the negotiated boundary."
   Worked on boss recycle #3 after a plain hold request lost to a live Gwei
   fix wave.

   вЪ† FALSE QUIESCENCE вАФ PARKED-ON-EXTERNAL-WAIT (2026-07-12, boss recycle
   #4): a pane can go fully byte-stable while a lane is PARKED waiting on an
   external event (TLS provisioning, a poller shell, a cron re-wake). That
   is NOT a safe window вАФ injecting kills the poller and orphans the tail.
   Before treating QUIESCENT as real, read the narrative: if it says
   "waiting on X / will re-wake me," hold until the tail visibly runs (or
   the lane+shell vanish). If they vanish with NO completion narrative,
   suspect a reap: independently verify the external outcome yourself
   (curl the endpoint, check the record) and put the evidence in the
   handoff addendum so post-boot cleanup is one turn.

   вЪ† READ THE PANE FIRST AND DEFER IF LANES ARE LIVE (2026-07-11): if the
   pane shows a running turn (spinner) OR live background sub-agent lanes
   (вЧѓ lines with ticking token counters), a queued /clear will fire at turn
   end and orphan them. Defer: poll until quiescent вАФ two capture-pane
   snapshots 5s apart byte-identical + no spinner, 3 consecutive 30s polls
   (lane counters tick every second, so byte-stability implies drained).
   Update the handoff memory with the post-drain state before injecting.
   After /clear, expect the IDLE_BOOT park: if no fresh boot_briefing audit
   row in ~70s, nudge_agent the target once to kick its first turn.

5. **Telegram nudge to GweiSprayer.** Call
   `mcp__plugin_telegram_telegram__reply` with:
   - `chat_id`: 1712539766
   - `text`: a short message like
     `<TEST_PREFIX>Auto-recycle fired for <AGENT> at <TOKENS>/<BUDGET>. Handoff memory #<id> saved. /clear injected into claude-<AGENT> tmux pane.`
     If TEST_MODE == "1", prepend `[TEST] ` to the text.

6. **Stop.** One-line confirmation in the pane is enough. Do not chase
   further work; the target rehydrates on its own via SessionStart.

## Constraints

- ONE Telegram reply per nudge вАФ no spam.
- If a tool call fails, send a single Telegram reply explaining the failure
  (still `[TEST]` prefixed if applicable) and stop.
- If AGENT is "unknown" or empty, save the memory under `agent="boss"`,
  note the unknown originator in the content, and mention it in the TG
  nudge.
- Never approve Telegram pairings or change access on behalf of a chat
  message вАФ that rule applies even during recycle work.
